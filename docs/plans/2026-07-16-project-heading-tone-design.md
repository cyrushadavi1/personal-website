# Project Heading Tone Design

## Goal

Make both project case studies easier to scan by replacing playful, suspenseful,
or conversational headings with direct descriptions of the engineering topic.

## Scope

- Revise both project taglines.
- Revise every level-two heading in both project case studies.
- Keep project names, body copy, metadata, layout, and visuals unchanged.

## Writing standard

- Name the system, decision, failure mode, or capability discussed in the section.
- Prefer concrete technical language over jokes, narrative reveals, or commands.
- Keep headings concise enough for the desktop table of contents.
- Preserve the existing lowercase heading style.
- Use parallel phrasing where sections serve similar purposes, such as
  `current capabilities` and `planned improvements`.

## Validation

- Confirm every project tagline and level-two heading is updated.
- Build the Astro site and run the existing test suite.
- Check that the longer labels wrap cleanly in desktop and mobile navigation.
